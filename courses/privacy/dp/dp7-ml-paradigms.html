Where to Inject Noise?

Three training-time strategies for DP-ERM.

Three Training-Time Strategies

1. Parameter / Output

Solve $w^* = \arg\min L(w; D)$ exactly; release $w^* + Z$ with calibrated $Z$. Works for simple, strongly convex losses with bounded global sensitivity.

2. Objective

Add a random linear term: $L(w; D) + \langle b, w\rangle$; fully optimize the noisy objective. Yields a private optimum analytically.

3. Gradient (DP-SGD)

Clip per-example gradients, add Gaussian noise, account privacy across steps. Default for deep learning.

For convex losses, all three give excess risk $\;O\bigl(\tfrac{d \log(1/\delta)}{n\varepsilon}\bigr)$.
02

Where to Draw the Privacy Boundary?

Input, inference, or the model itself.

Input Perturbation — Local DP

Setup.

Noise is added to raw data $z_i$ before any model sees it (e.g. randomized response).

  • Strongest trust model: server never sees real data.
  • Severe utility loss in high-dimensional tasks (vision, NLP) — noise destroys signal.
  • Discussed in detail in the DP-FL deck (Lecture 8).

Inference DP — One-Off Output Perturbation

Train $f_\theta$ non-privately; at inference release $f_\theta(x) + Z$ with $Z$ scaled to global sensitivity.

Critical flaw.

Composition: $k$ queries cost $\approx k\varepsilon$. After enough queries, the privacy budget is exhausted and the model must be shut down.

Unsuitable for public-facing or high-throughput APIs.

Model DP (Global DP) — The Goal

Idea.

Bake the privacy guarantee into the model parameters $\theta$ (DP-SGD, PATE-trained students).

  • By post-processing immunity, the deployed model can be queried infinitely without further leakage.
  • Avoids the rapid budget exhaustion of inference DP and the utility loss of local DP.
  • Modern DP-ML targets Model DP.
03

PATE

Private Aggregation of Teacher Ensembles.

PATE — Core Intuition

DP-SGD interferes with the optimizer; PATE shifts the paradigm:

If many independent models agree on a prediction, that prediction does not depend on any single sensitive record.

PATE replaces gradient perturbation with ensemble learning + noisy aggregation + knowledge transfer.

Papernot, Abadi, Erlingsson, Goodfellow, Talwar, "Semi-supervised Knowledge Transfer for Deep Learning from Private Training Data", ICLR 2017.

Phase 1 — Teacher Ensemble

  • Partition the sensitive dataset $X = \bigcup_{i=1}^n X_i$ into disjoint subsets.
  • Train teacher $T_i$ on $X_i$ independently. Architectures may differ.

Privacy property.

Adding/removing one individual changes exactly one subset $X_i$ — and thus exactly one teacher $T_i$.

Phase 2 — Noisy Aggregation of Votes

For a public input $x$, each teacher casts a vote $y_i = T_i(x) \in \mathcal{Y}$. Tally:

$$v_y \;=\; |\{i : T_i(x) = y\}| \qquad \text{for each } y \in \mathcal{Y}.$$
  • Laplace PATE: $\tilde v_y = v_y + \mathrm{Lap}(1/\varepsilon)$.
  • PATE-G (Gaussian): $\tilde v_y = v_y + \mathcal{N}(0, \sigma^2)$.
  • Aggregated label: $\hat y = \arg\max_y \tilde v_y$ (a noisy-max query, sensitivity 1).

Phase 3 — Train a Student Model

Problem.

If end-users query the teacher ensemble directly, every query consumes privacy budget — Inference-DP trap.

Solution.

Train a Student on the public-data examples labeled by the noisy ensemble. Deploy the student.

By post-processing immunity, the student is Model DP — query it forever.

PATE Pipeline

Sensitive data $X = \bigcup X_i$
→
Teachers $T_1, \dots, T_n$
→
Vote $v_y$ + noise
Public unlabeled $x^{\mathrm{pub}}$
→
Noisy labels $\hat y$
→
Student (Model DP)
Only the student is deployed. The teacher ensemble — and the sensitive data — never leave the trusted boundary.
04

Accounting & Trade-offs

Composition, advantages, and limitations.

Privacy Accounting

Labeling $m$ public examples requires composing $m$ noisy-aggregation queries. Advanced composition:

$$\varepsilon' \;=\; \sqrt{2 m \log(1/\delta')}\, \varepsilon \;+\; m\, \varepsilon (e^\varepsilon - 1).$$
  • Scales poorly for moderate $m$.
  • Modern PATE uses Rényi DP: when teachers strongly agree (large vote margin), the per-query RDP cost is much smaller than worst-case — data-dependent accounting.
RDP exploits consensus: high-agreement queries cost much less than the worst-case bound suggests.

PATE — Advantages

  • Architecture-agnostic. Teachers can be SVMs, decision trees, or NNs; student can be a different architecture entirely.
  • Intuitive guarantee. One person affects one teacher; consensus implies robustness.
  • Parallelizable. Teachers train independently; no cross-batch communication.
  • Inherits post-processing. Deployed student is Model DP.

PATE — Limitations

  • Needs a public unlabeled dataset in the same domain as the sensitive data.
  • Needs a large sensitive dataset: $n$ subsets must each be big enough to train an accurate teacher.
  • Scales poorly with $|\mathcal{Y}|$: noisy max over many classes spreads noise — degraded utility for $|\mathcal{Y}|$ in the hundreds.

Bottom line.

PATE shines when teachers actually agree. When agreement is rare (long-tail classes, distribution shift), DP-SGD typically wins.

Recap

  • Three perturbation strategies for DP-ERM: parameter, objective, gradient.
  • Privacy boundary: input (local DP), inference (composes badly), model (the goal).
  • PATE: teacher ensemble → noisy aggregation → student. Model DP via post-processing.
  • Accounting: RDP makes data-dependent (consensus-aware) bounds practical.
