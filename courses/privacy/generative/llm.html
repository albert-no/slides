Tokens and Sequences

Tokenizer maps text to integer IDs (BPE, SentencePiece).

$$\text{text} \;\longmapsto\; x_{1:T} = (x_1, \ldots, x_T) \in \mathcal{V}^T,\qquad |\mathcal{V}| \approx 5\!\times\!10^4.$$
  • $\mathcal{V}$: fixed vocabulary (subword units)
  • Training, sampling, loss — all per-token

Every privacy quantity in this course is token-level.

Autoregressive Factorization

Chain rule on a discrete sequence:

$$p(x_{1:T}) \;=\; \prod_{t=1}^{T} p(x_t \mid x_{<t}),\qquad x_{<t} := (x_1, \ldots, x_{t-1}).$$

An LLM is a parameterized next-token distribution:

$$p_\theta(\,\cdot \mid x_{<t}\,) \;\in\; \Delta^{|\mathcal{V}|-1}.$$

The whole model collapses to one object: $p_\theta(\cdot \mid \text{prefix})$.

Architecture

Decoder-only transformer, logits, softmax

Decoder-Only Transformer

  • Stack of $N$ blocks:
    masked self-attention + FFN,
    each wrapped in residual + LayerNorm
  • Causal mask:
    position $t$ attends only to $x_{<t}$
  • $z_t \in \mathbb{R}^{|\mathcal{V}|}$ at every position in one forward pass
    — softmax gives $p_\theta(\cdot\mid x_{<t})$
  • Decoder-only since GPT-2
    — drop the encoder, keep the right half
Vaswani et al., "Attention Is All You Need", NeurIPS 2017; Radford et al., GPT-2, OpenAI 2019.

Logits to Next-Token Distribution

Softmax over the vocabulary:

$$p_\theta(v \mid x_{<t}) \;=\; \frac{\exp(z_{t,v})}{\sum_{v' \in \mathcal{V}} \exp(z_{t,v'})}, \qquad v \in \mathcal{V}.$$
  • Training: all $T$ conditionals computed in parallel (causal mask)
  • Inference: sample $\hat x_t$, append, repeat

Logits $z_t$ and probabilities $p_\theta(\cdot\mid x_{<t})$ are the privacy signals.

Sequential Generation

Training is parallel; inference is sequential.

  • Step $t$: forward pass on $x_{<t}$ → $p_\theta(\cdot\mid x_{<t})$ → sample $\hat x_t$ → append
  • One full forward pass per token (KV-cache reuses past activations)
  • $O(T)$ sequential steps — the inference-latency bottleneck
03

Training

Pretraining NLL, scale, alignment

Pretraining Objective

Maximum likelihood on corpus $\mathcal{D}$ — sum of per-token cross-entropies:

$$\mathcal{L}(\theta) \;=\; -\,\frac{1}{|\mathcal{D}|}\sum_{x \in \mathcal{D}}\;\sum_{t=1}^{T} \log p_\theta(x_t \mid x_{<t}).$$
  • Each token position contributes one loss term
  • Per-token loss $\ell_t(x;\theta) = -\log p_\theta(x_t \mid x_{<t})$ is a core observable

Low $\ell_t$ on a member sample $\Rightarrow$ MIA / memorization signal.

Scale Makes Memorization Real

Corpus

$\sim 10^{12}\text{–}10^{13}$ tokens.

Web scrape, books, code, dialogues.

Parameters

$\sim 10^{10}\text{–}10^{12}$.

Capacity to store rare strings verbatim.

Pretraining data does resurface — extractable, attackable, suable.

Hoffmann et al., "Training Compute-Optimal LLMs" (Chinchilla), NeurIPS 2022.
Carlini et al., "Quantifying Memorization Across Neural Language Models", ICLR 2023.

Post-Training I — Supervised Fine-Tuning

Align the pretrained model on curated $(x, y)$ pairs — prompt and demonstration response.

  • Data: human demonstrations or distillation from stronger models
  • Loss: per-token NLL, masked to response tokens only
  • Effect: shifts surface behavior; pretraining memory stays in the weights
  • Output: reference model $\pi_\text{ref}$ for the next stage
$$\mathcal{L}_{\text{SFT}}(\theta) \;=\; -\!\!\sum_{(x,y)\,\in\,\mathcal{D}_{\text{SFT}}}\;\sum_{t=1}^{|y|} \log p_\theta(y_t \mid x,\, y_{<t}).$$
Ouyang et al., InstructGPT, NeurIPS 2022.

Post-Training II — Preference Optimization

Move the model from imitating demonstrations to matching preferences.

  • Data: triples $(x,\,y_w,\,y_l)$ — prompt, preferred $y_w$, dispreferred $y_l$
  • Goal: prefer $y_w$ over $y_l$, while staying close to $\pi_\text{ref}$ (the SFT model)
  • Classic RLHF: learn reward $r_\phi$, then PPO with KL anchor to $\pi_\text{ref}$
  • DPO: closed-form solution to the same problem — direct loss on $\pi_\theta$, no reward model

Pretraining sets capability; SFT + preference optimization sets behavior.

Christiano et al., NeurIPS 2017; Rafailov et al., DPO, NeurIPS 2023.

DPO Loss

$$\mathcal{L}_{\text{DPO}}(\theta) \;=\; -\,\mathbb{E}_{(x,y_w,y_l)}\!\left[\,\log \sigma\!\left(\beta \log \tfrac{\pi_\theta(y_w\mid x)}{\pi_\text{ref}(y_w\mid x)} \;-\; \beta \log \tfrac{\pi_\theta(y_l\mid x)}{\pi_\text{ref}(y_l\mid x)}\right)\right]$$
  • $\pi_\text{ref}$: frozen reference. $\beta$: KL-regularization strength.
  • Implicit reward $\;r_\theta(x,y) = \beta \log \pi_\theta(y\mid x)/\pi_\text{ref}(y\mid x)$
  • Push $y_w$ up, push $y_l$ down — only log-ratios relative to $\pi_\text{ref}$ matter

Preview (unlearning): NPO keeps only the $y_l$ term — DPO without preferred examples, on a forget set.

Rafailov et al., DPO, NeurIPS 2023.

Sampling and Privacy Hooks

Where memorization, MIA, watermark, unlearning enter

Sampling — Temperature

Logit $z_{t,v} \in \mathbb{R}$ — pre-softmax score per vocab item.

$$p^{(\tau)}(v\mid x_{<t}) \;\propto\; \exp(z_{t,v}/\tau),\qquad \tau > 0.$$
  • $\tau \to 0^+$: greedy, $\hat x_t = \arg\max_v z_{t,v}$ — sharpest, deterministic
  • $\tau = 1$: the model's own distribution
  • $\tau \to \infty$: uniform over $\mathcal{V}$ — flattest

Temperature reshapes the same logits — no weight change.

Sampling — Truncation

Cut the long tail before sampling.

  • Top-$k$: keep $k$ largest logits, renormalize, sample
  • Top-$p$ (nucleus): smallest set $S \subseteq \mathcal{V}$ with $\sum_{v\in S} p(v) \ge p$, sample within $S$
  • Top-$k$: fixed support size; top-$p$: fixed mass — top-$p$ adapts to entropy
  • Combined with temperature in practice

All sampling rules are post-hoc transforms of $p_\theta$ — watermarks live here.

Holtzman et al., "The Curious Case of Neural Text Degeneration", ICLR 2020.
