01

Motivating Attacks

Aggregate answers can still expose individuals.

A Single Sensitive Query

A company offers health insurance and queries the insurer:

"How many employees born on [date], living in [ZIP], suffer from [condition]?"
  • If the answer is 1, and only one employee fits the (date, ZIP), the company learns that person's diagnosis.
  • Even "fewer than 5" still leaks information when combined with other queries.

Difference Attack

Two near-duplicate queries can isolate one record:

  1. How many employees joined before date $t$ and have the condition?
  2. How many employees joined before date $t+1$ and have the condition?
$$a_2 - a_1 \in \{0,1\} \;\Longrightarrow\; \text{status of the single employee who joined on day } t.$$
Aggregate statistics can hide nothing if the adversary controls the query set.

Reconstruction Attack

Definition (Reconstruction Attack).

An attacker is given aggregate statistical query responses $a_1, \dots, a_m$ (possibly noisy) and infers most or all individual entries of the private dataset $D$.

DP emerges from a tight question: how accurately can we answer many queries while preventing reconstruction?

02

Formal Setup

Binary datasets, counting queries, bounded-error answers.

Setup: Binary Dataset and Counting Queries

  • Dataset. $D = (d_1, \dots, d_n) \in \{0,1\}^n$; $d_i$ marks whether individual $i$ has a sensitive attribute.
  • Counting query. For $S \subseteq \{1,\dots,n\}$, $\,q_S(D) = \sum_{i \in S} d_i$.
  • Curator answers. $a_1, \dots, a_m$ with $|a_j - q_{S_j}(D)| \le E$ for all $j$.
  • Adversary model. Answers can be random or chosen adversarially within the error bound.

The Reconstruction Optimization

Given $(S_j, a_j)$, the attacker solves:

$$\hat D \;=\; \arg\min_{D \in \{0,1\}^n} \;\max_{1 \le j \le m} |q_{S_j}(D) - a_j|.$$
  • Exhaustive search: $2^n$ candidates → exponential complexity.
  • Returns the candidate that is most consistent with the (noisy) answers.
A polynomial-time attack via LP relaxation will appear later in the deck.

Trivial Consistency Bound

Claim.

If every query is answered to within error $\le \alpha n$, i.e. $\max_S |q_S(D) - a_S| \le \alpha n$, then the reconstruction $\hat D$ satisfies

$$\max_S |q_S(\hat D) - a_S| \;\le\; \alpha n.$$

The true $D$ is feasible, so the minimizer is at least as good. The work is in turning consistency into a closeness bound on $\hat D$.

03

Dinur–Nissim Theorems

Quantitative limits on aggregate-noise privacy.

Theorem 1 — All Queries, Bounded Error

Theorem (Dinur–Nissim 2003).

If every subset query is answered with error at most $\alpha n$, then the reconstruction error of $\hat D$ is at most $4\alpha n$:

$$\sum_{i=1}^n |D_i - \hat D_i| \;\le\; 4\alpha n.$$
Dinur & Nissim, "Revealing Information While Preserving Privacy", PODS 2003.

Proof of Theorem 1

Proof.

Define $A_{01} = \{i : \hat D_i = 0,\, D_i = 1\}$ and $A_{10} = \{i : \hat D_i = 1,\, D_i = 0\}$.

If $\hat D$ and $D$ disagree on $> 4\alpha n$ bits, WLOG $|A_{10}| > 2\alpha n$. Then

$$|q_{A_{10}}(D) - q_{A_{10}}(\hat D)| \;=\; |A_{10}| \;>\; 2\alpha n.$$

By triangle inequality and the per-query error bound:

$$|q_{A_{10}}(D) - q_{A_{10}}(\hat D)| \;\le\; |q_{A_{10}}(D) - a_{A_{10}}| + |a_{A_{10}} - q_{A_{10}}(\hat D)| \;\le\; 2\alpha n,$$

a contradiction. So disagreement $\le 4\alpha n$. $\blacksquare$

The attacker does not need to know $A_{10}$ — answers to all subsets are available.

Corollary: 1%-Error → 96%-Recovery

Corollary.

If all queries have error at most $\tfrac{1}{100}n$, then the reconstruction $\hat D$ matches $D$ on at least $\tfrac{96}{100} n$ entries.

Take-away: Worst-case error per query of $o(n)$ across all subsets is fatal — most records are recovered.

Assumption — answering every one of the $2^n$ subset queries — is unrealistic. Next: relax to polynomially many random queries.

Realistic Regime: $m = 20 n$ Random Queries

  • The attacker chooses $S_1, \dots, S_m$ uniformly at random, $m = 20n$.
  • Same reconstruction $\hat D = \arg\min_D \max_j |q_{S_j}(D) - a_j|$.
  • Curator guarantees $\max_j |q_{S_j}(D) - a_j| \le \alpha n$.

The constants 20 and 256 (next slide) are convenient; tighter constants exist.

Theorem 2 — Polynomially Many Random Queries

Theorem (Dinur–Nissim 2003).

With $m = 20n$ uniformly random queries each answered to within $\alpha n$, with probability $\ge 1 - 2^{-n}$ the reconstruction error obeys

$$\sum_i |D_i - \hat D_i| \;\le\; 256\, \alpha^2 n^2.$$

Meaningful when $\alpha \ll 1/\sqrt n$.

Proof Outline — Bad Set $B$

Let $B = \{\hat D : \hat D \text{ and } D \text{ disagree on } \ge 256\alpha^2 n^2 \text{ coords}\}$.

For any $D' \in B$, if $|q_{S_j}(D) - q_{S_j}(D')| \ge 4\alpha n$ then $D'$ cannot be the reconstruction:

$$|q_{S_j}(D') - a_j| \;\ge\; |q_{S_j}(D') - q_{S_j}(D)| - |q_{S_j}(D) - a_j| \;\ge\; 3\alpha n.$$

Goal: show with high probability every $D' \in B$ has some $j$ with $|q_{S_j}(D) - q_{S_j}(D')| \ge 4\alpha n$.

Proof — Random Subset Concentration

Let $Z = D - D_0 \in \{-1, 0, 1\}^n$ for $D_0 \in B$; $Z$ has $\ge 256 \alpha^2 n^2$ nonzero entries.

For a uniformly random subset $S_j$:

$$\Pr\bigl[|q_{S_j}(D) - q_{S_j}(D_0)| \le 4\alpha n\bigr] \;\le\; \tfrac{9}{10}.$$

(Exercise: variance bound + Chebyshev.)

By independence across $j$:

$$\Pr\bigl[\forall j,\; |q_{S_j}(D) - q_{S_j}(D_0)| \le 4\alpha n\bigr] \;\le\; (9/10)^m \;\le\; 2^{-2n}.$$

Union bound over $|B| \le 2^n$:

$$\Pr\bigl[\exists D_0 \in B \text{ surviving}\bigr] \;\le\; 2^{-2n} \cdot 2^n \;=\; 2^{-n}. \quad\blacksquare$$
04

Efficient Attack

From exponential search to a polynomial LP.

Linear-Programming Relaxation

Relax $D \in \{0,1\}^n$ to $D \in [0,1]^n$:

$$\tilde D \;=\; \arg\min_{D \in [0,1]^n} \;\max_j |q_{S_j}(D) - a_j|,$$

then round each $\tilde D_i$ to $\{0,1\}$ to obtain $\hat D$.

  • The relaxed problem is a linear program in $n$ variables and $m$ constraints — solvable in polynomial time.
  • Rounding only inflates the error by a constant, preserving the Dinur–Nissim guarantees.

Linear Program (Refresher)

Definition (Linear Program).

Given $c \in \mathbb{R}^d$, $a_1, \dots, a_m \in \mathbb{R}^d$, and $b_1, \dots, b_m \in \mathbb{R}$:

$$\max_{x \in \mathbb{R}^d} \; c \cdot x \quad\text{s.t.}\quad a_i \cdot x \le b_i, \; 1 \le i \le m.$$

LPs can be solved in polynomial time (simplex, interior-point). The reconstruction min–max recasts into LP form by introducing a slack variable for the max.

Implications

  • "Too accurate" aggregates leak. Sub-$\sqrt n$ per-query error suffices for catastrophic reconstruction.
  • Noise calibration is essential. Privacy versus accuracy is a fundamental, quantitative trade-off.
  • Worst-case adversary. The attacker chooses both queries and exploits any error pattern.
These limits motivate Differential Privacy: a rigorous, composable framework for noise calibration — the subject of the next deck.
